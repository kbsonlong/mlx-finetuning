# Autoresearch + MLX-LoRA 自学习微调方案

> **创建日期**: 2026-03-30
> **版本**: 1.0
> **目标**: 集成 autoresearch-macos 和 mlx-lora，实现自主迭代的 LLM 微调系统

---

## 一、项目概述

### 1.1 核心理念

将 autoresearch 的**自主研究循环**与 mlx-lora 的**高效微调能力**结合，创建一个能够在 Apple Silicon Mac 上自动优化 LoRA 微调参数的智能系统。

### 1.2 系统架构

```
                    ┌─────────────────────────────────────────┐
                    │         AI Agent (Claude/GPT)           │
                    │        (研究主管 + 实验执行者)           │
                    └─────────────────┬───────────────────────┘
                                      │
                    ┌─────────────────▼───────────────────────┐
                    │          研究循环控制器                  │
                    │    (program.md 指令集 + 状态管理)        │
                    └─────┬───────────────────────────────┬───┘
                          │                               │
         ┌────────────────┴───────┐           ┌──────────┴──────────┐
         │   数据生成模块          │           │   微调实验模块       │
         │  (DeepSeek蒸馏+增强)   │           │  (MLX-LoRA训练)      │
         └────────────────────────┘           └─────────────────────┘
                          │                               │
         ┌────────────────┴───────────────────────────────┴──────┐
         │              评估与决策模块                          │
         │  (验证集评估 → 指标比较 → 保留/丢弃 → 记录结果)         │
         └────────────────────────────────────────────────────────┘
```

## 二、两个项目的核心能力分析

### 2.1 autoresearch-macos 核心能力

| 能力 | 说明 | 价值 |
|------|------|------|
| **自主实验循环** | AI 修改代码 → 运行 → 评估 → 决策 | 实现自动化研究 |
| **固定时间预算** | 每次实验固定 5 分钟 | 结果可比较 |
| **Git 版本管理** | 自动 commit/reset | 完整实验历史 |
| **结果日志** | TSV 格式记录所有实验 | 可追溯分析 |
| **改进保留** | val_bpb 降低则保留 | 单调优化 |

### 2.2 mlx-lora 核心能力

| 能力 | 说明 | 价值 |
|------|------|------|
| **Apple Silicon 优化** | MLX 框架原生支持 | 高效本地训练 |
| **LoRA 微调** | 低资源高效微调 | 适合个人开发 |
| **DeepSeek 蒸馏** | 生成高质量训练数据 | 数据质量保障 |
| **数据增强** | 上下文/复杂度/多轮 | 数据多样性 |
| **K8S 领域特化** | 垂直领域专家 | 实用价值高 |

## 三、集成方案设计

### 3.1 核心设计原则

```
┌─────────────────────────────────────────────────────────────┐
│                     设计原则                                │
├─────────────────────────────────────────────────────────────┤
│  1. 分离关注点：数据生成 vs 微调参数研究                     │
│  2. 固定数据：实验阶段使用相同数据集                         │
│  3. 可调参数：LoRA rank/alpha, lr, batch_size, epochs       │
│  4. 评估指标：验证集 loss/accuracy/perplexity               │
│  5. 时间预算：每次实验 10-15 分钟 (MLX 比 PyTorch 慢)       │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 项目结构设计

```
mlx-autoresearch/
├── README.md
├── pyproject.toml           # 依赖管理
│
├── prepare.py               # 数据准备 (一次性运行)
│   ├── DeepSeek 蒸馏数据生成
│   ├── 数据增强
│   └── MLX 格式转换
│
├── train.py                 # 微调训练脚本 (AI 修改目标)
│   ├── LoRA 配置
│   ├── 训练超参数
│   └── 评估输出
│
├── program.md               # AI Agent 指令集 (人编辑)
│   ├── 实验设置指南
│   ├── 参数调整策略
│   └── 决策规则
│
├── scripts/
│   ├── distill_data.py      # DeepSeek 蒸馏
│   ├── augment_data.py      # 数据增强
│   └── evaluate_adapter.py  # 适配器评估
│
├── config/
│   ├── base_config.yaml     # 基础配置 (固定)
│   └── model_path.yaml      # 模型路径 (本地配置)
│
└── outputs/
    ├── adapters/            # LoRA 适配器输出
    ├── logs/                # 训练日志
    └── results.tsv          # 实验结果记录
```

### 3.3 可调参数空间

```python
# AI 可以探索的参数空间
SEARCH_SPACE = {
    # LoRA 结构参数
    "lora_rank": [4, 8, 16, 32],
    "lora_alpha": [8, 16, 32, 64],
    "lora_dropout": [0.0, 0.05, 0.1],

    # 训练超参数
    "learning_rate": [1e-5, 3e-5, 5e-5, 1e-4],
    "batch_size": [2, 4, 8],
    "iters": [50, 100, 200],
    "max_seq_length": [256, 512, 1024],

    # 优化器配置
    "optimizer": ["adam", "adamw"],
    "grad_checkpoint": [True, False],

    # 学习率调度
    "lr_schedule": ["constant", "cosine", "linear"],
}
```

### 3.4 评估指标设计

```python
# 主评估指标 (越小越好)
METRICS = {
    "val_loss": "验证集交叉熵损失 (主要优化目标)",

    # 辅助指标
    "train_loss": "训练集损失 (检测过拟合)",
    "val_accuracy": "验证集准确率",
    "perplexity": "困惑度 (exp(val_loss))",
    "memory_mb": "显存占用 (资源约束)",
    "time_seconds": "训练时间 (效率)",
}
```

## 五、实验循环流程

### 5.1 完整实验循环

```
┌─────────────────────────────────────────────────────────────────┐
│                        实验循环开始                              │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ 1. 当前状态分析                                                  │
│    ├─ 查看 Git commit/分支                                       │
│    ├─ 读取 results.tsv (历史结果)                               │
│    └─ 确定当前最佳配置                                          │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. 生成实验假设                                                  │
│    ├─ 基于历史结果分析                                          │
│    ├─ 选择参数调整方向                                          │
│    └─ 定义具体修改 (如: 增加 lora_rank 8→16)                   │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. 修改训练代码                                                  │
│    ├─ 编辑 train.py 中的超参数                                  │
│    ├─ Git commit (带描述性消息)                                 │
│    └─ 记录实验意图                                              │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. 执行训练                                                      │
│    ├─ 运行: python train.py > run.log 2>&1                     │
│    ├─ 监控训练过程                                              │
│    └─ 等待完成 (超时保护)                                      │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ 5. 提取结果                                                      │
│    ├─ 解析 run.log 获取评估指标                                 │
│    ├─ val_loss, val_accuracy, perplexity                       │
│    └─ 资源使用 (内存, 时间)                                     │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ 6. 决策                                                          │
│    ├─ if val_loss < best_val_loss:                             │
│    │   保留 (保留 commit, 更新最佳)                             │
│    ├─ else:                                                     │
│    │   丢弃 (git reset)                                         │
│    └─ 记录到 results.tsv                                        │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│ 7. 返回步骤 1 (循环直到手动停止)                                │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 program.md 指令模板

```markdown
# MLX-LoRA 自主微调研究

## 实验设置

1. **数据集**: 固定使用 `data/mlx_lm/` 中已准备的数据
2. **基础模型**: `models/Qwen3.5-9B-MLX-4bit/`
3. **时间预算**: 每次实验 10 分钟
4. **评估指标**: val_loss (越低越好)

## 可修改范围

你**只能**修改 `train.py` 中的以下部分:

- LoRA 配置: rank, alpha, dropout
- 训练超参数: learning_rate, batch_size, iters
- 序列长度: max_seq_length
- 优化器: optimizer, lr_schedule

## 实验循环

1. 分析当前 Git 状态
2. 基于 results.tsv 历史提出改进假设
3. 修改 train.py 并 commit
4. 运行训练: `python train.py > run.log 2>&1`
5. 提取结果: `grep "val_loss:" run.log`
6. 决策:
   - 改进 → 保留 commit
   - 退步 → git reset
7. 记录到 results.tsv

## 决策规则

- **主指标**: val_loss (降低 = 好)
- **约束**: 内存 < 16GB
- **简化原则**: 相近性能优先简单配置

## 持续运行

**永不停止**。持续实验直到用户手动中断。
```

## 七、风险与挑战

### 7.1 技术挑战

| 挑战 | 影响 | 缓解方案 |
|------|------|----------|
| MLX 生态不成熟 | 中 | 依赖稳定版本，预留回退 |
| 本地资源限制 | 中 | 合理设置 batch_size |
| 评估噪声 | 高 | 多次运行取平均 |
| 过拟合风险 | 中 | 早停 + 正则化 |

### 7.2 实施建议

1. **渐进式迭代**: 先实现核心功能，再逐步完善
2. **充分测试**: 每个阶段完成后进行端到端测试
3. **文档同步**: 代码和文档保持同步更新
4. **社区反馈**: 及时关注 MLX 和相关项目更新
